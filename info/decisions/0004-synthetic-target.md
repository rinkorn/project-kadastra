# ADR-0004: Синтетический таргет для пилота

**Статус:** Accepted
**Дата:** 2026-04-25

## Контекст

Для обучения модели оценки нужна целевая переменная (стоимость ₽/м² или стоимость объекта). В нашем бакете `s3://kadastrova/Kadatastr/` таргета нет — см. [data-sources.md](../data-sources.md). Реальные источники (ПКК Росреестра, Avito/Cian, региональные опендата) на пилот не собираются — по времени и лицензиям.

Варианты:

- **A. Ждать реальный таргет** — пилот стоит, нечего обучать.
- **B. Синтетический proxy** — формула от расстояния до центра Казани + буст от метро и застройки + гауссов шум. Модель учится предсказывать эту формулу; смысл — проверить ML-пайплайн (spatial CV, MLflow, инференс на карту) до появления реального таргета.
- **C. Обучаться без таргета** (unsupervised/anomaly) — не наш кейс, нужна регрессия стоимости.

## Решение

**B. Синтетический proxy-таргет.**

Формула (в [src/kadastra/etl/synthetic_target.py](../../src/kadastra/etl/synthetic_target.py)):

```text
target = BASE * exp(-dist_km / DECAY_KM)
       * (1 + METRO_BOOST * 1[count_stations_1km > 0])
       * (1 + BUILDINGS_BOOST * log1p(building_count))
       + noise
```

Параметры:

| Константа | Значение | Смысл |
|-----------|----------|-------|
| `BASE_PRICE_RUB_PER_M2` | 80 000 | Цена в «нулевой точке» (центр Казани, без бустов) |
| `DECAY_KM` | 30 | Характерная длина экспоненциального спада |
| `METRO_BOOST` | 0.5 | +50% если в гексе есть хоть 1 станция в 1 км |
| `BUILDINGS_BOOST` | 0.1 | Логарифмический буст от плотности застройки |
| `NOISE_SIGMA` | 5000 | σ гауссова шума в ₽/м² |

Seed — `synthetic_target_seed: int = 42` в `Settings`. Дистанция считается векторизованным haversine'ом от центра Казани (55.7887, 49.1221) до центра каждой H3-ячейки (`h3.cell_to_latlng`). Отрицательные значения после шума клипаются к 0.

Обязательные входные колонки: `h3_index`, `resolution`, `building_count`, `count_stations_1km`. При отсутствии — `KeyError`.

## Обоснование выбора B vs A/C

- A останавливает пилот; мы хотим закрыть ML-трек.
- C не даёт того, что отображается на карте (стоимость — это регрессия).
- B даёт:
  - явно нелинейную зависимость от фичей (exp + log + флаг метро), которую CatBoost учит без тривиализации;
  - контроль истины: мы знаем формулу, можем проверить что модель сходится к ней и spatial CV ловит утечки через соседство;
  - готовую ось цвета для карты до появления реального таргета.

## Последствия

- Добавлены `synthetic_target_store_path: Path = Path("data/gold/targets")` и `synthetic_target_seed: int = 42` в `Settings`.
- Pure-функция `compute_synthetic_target(gold, *, seed) -> pl.DataFrame` добавляет колонки `kazan_distance_km` и `synthetic_target_rub_per_m2`.
- Use case `BuildSyntheticTarget(gold_reader, target_store, seed)` читает gold, применяет формулу, пишет в `data/gold/targets/region={r}/resolution={res}/data.parquet`.
- CLI — [scripts/build_synthetic_target.py](../../scripts/build_synthetic_target.py).
- Когда появится реальный таргет — делаем отдельный слой `data/gold/targets_real/...` и новый ADR; текущий синтетический не выкидываем, останется как baseline.

## Sanity check (zoom-out)

Прогон на Татарстане, res=8 (85 674 ячейки, seed=42):

| Метрика | Значение |
|---------|----------|
| kazan_distance_km min / median / max | 0.17 / 136.8 / 338.9 |
| synthetic_target_rub_per_m2 min / median / max | 0 / 2 690 / 177 479 |

Топ-5 самых дорогих гексов — все в радиусе ~3.5 км от центра Казани, с метро и сотнями зданий. Нижние — на границе Татарстана (~335 км), клипнуты к 0 отрицательным шумом. Ожидаемо: при `exp(-335/30) ≈ 1.5e-5` база 80к даёт ~1 ₽, шум σ=5000 доминирует.

## Открытые вопросы

- Использовать ли `road_length_m` в формуле (сейчас нет — чтобы модель не выучивала таргет тривиально через линейный фичер)?
- Оставить `DECAY_KM=30` или поднять (сейчас слишком быстро затухает — большая часть Татарстана около нуля)?
- Когда появится реальный таргет — решение вытеснения: держать оба как бенчмарк vs убрать синтетику.
